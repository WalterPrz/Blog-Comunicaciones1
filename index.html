2.1. Caso Europa

Una empresa posee tres sucursales en diferentes puntos del país. Dispone una casa matriz, en la cual debe instalar una Sala de Equipos, de tal manera de que le brinden servicios básicos al resto de sucursales haciendo uso de Internet para tales propósitos.

Posee la Casa Matriz en San Salvador con la estructura: Finanzas, Ventas y Producción. En cada sucursal dispone de la misma estructura que en la casa matriz, sólo que en menor proporción.

Requerimientos del caso

  1. Diseñar un esquema de dirección IP acorde a la empresa.
  2. Modelar la red en GNS3, un host por sucursal según corresponda.
  3. Diseñar un esquema de VLAN, que sea funcional en toda la empresa, y moldearlo en GNS3
  4. Establecer los protocolos de enrutamiento adecuados para la empresa.
  5. Establecer una DMZ, para levantar servicios a través de NAT.
  6. Montar los servicios web necesarios: Http y HTTPS en una máquina virtual, que acceda a aplicaciones dispuestas a través de un servicio de archivos NFS.
  7. Montar el servicio de DNS, en una máquina virtual, para la resolución de nombres de dominio: europa.net
  8. Establecer el servicio de DHCP según sea necesario en cada sucursal.
  9. Presentar un presupuesto detallado de costo real, así como características reales del equipamiento a utilizar en el despliegue de la red y sala de datos.

DHCP

El DHCP es una extensión del protocolo Bootstrap (BOOTP) desarrollado en 1985 para conectar dispositivos como terminales y estaciones de trabajo sin disco duro con un Bootserver, del cual reciben su sistema operativo.

El DHCP se desarrolló como solución para redes de gran envergadura y ordenadores portátiles y por ello complementa a BOOTP, entre otras cosas, por su capacidad para asignar automáticamente direcciones de red reutilizables y por la existencia de posibilidades de configuración adicionales.

HTTPS

(protocolo de Transferencia de Hiper-Texto) es un protocolo que permite establecer una conexión segura entre el servidor y el cliente, que no puede ser interceptada por personas no autorizadas. En resumidas cuentas, es la versión segura de el http (Hyper Text Transfer Protocol)

Una conexión HTTP estándar en Internet puede ser fácilmente secuestrada por partes no autorizadas. El propósito de una conexión HTTPS es evitar esto: encriptar los datos para asegurar una transmisión de datos segura. La transmisión está encriptada y el servidor autenticado.

NFS

El sistema de archivos de red (Network File System, NFS) es una aplicación cliente/servidor que permite a un usuario de equipo ver y, opcionalmente, almacenar y actualizar archivos en un equipo remoto como si estuvieran en el propio equipo del usuario. El protocolo NFS es uno de varios estándares de sistema de archivos distribuidos para almacenamiento atado a la red (NAS).

NFS permite al usuario o administrador del sistema montar (designar como accesible) todo o una porción de un sistema de archivos en un servidor. La parte del sistema de archivos que se monta puede ser accedida por los clientes con los privilegios que se asignan a cada archivo (de sólo lectura o de lectura y escritura)
