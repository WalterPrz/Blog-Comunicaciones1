Pasos para crear los servidores:

Las diferentes configuraciones las hemos realizado en una máquina virtual con Debian 11, para configurar todo el entorno de trabajo para que funcione correctamente el Caso de Estudio dee Europa.

Para cada una de estas secciones es necesario realizar antes una actualizacion de paquetes.

sudo apt-get update

Establecer una ip estatica para el servidor.

Nano /etc/network/interfaces

Dejamos el archivo asi:

Reiniciamos

ifdown -a; ifup -a;

instalamos:

sudo apt-get install -y bind9 bind9utils bind9-doc dnsutils

Autorizamos conexiones por el firewall

sudo ufw allow dns

creamos las zonas

sudo nano /etc/bind/named.conf.local

Zona directa

Zona indirecta

Copie las entradas de muestra en el archivo de zona llamado forward.europa.net.db para la zona de avance en el directorio / etc / bind

sudo cp /etc/bind/db.local /etc/bind/forward.europa.net.db

editamos la zona

sudo nano /etc/bind/forward.europa.net.db

Zona Directa

Copie las entradas de muestra en el archivo de zona llamado reverse.europa.net.db para la zona inversa en el directorio / etc / bind y cree punteros inversos para los registros de la zona de avance anteriores.

sudo cp /etc/bind/db.127 /etc/bind/reverse.europa.net.db

editamos la zona

sudo nano /etc/bind/reverse.europa.net.db

Zona Inversa

Validamos la sintaxis del archi de configuración BIND.

sudo named-checkconf

Activar Zona Directa

sudo named-checkzone europa.net /etc/bind/forward.europa.net.db

Activar Zona Inversa

sudo named-checkzone 1.168.192.in-addr.arpa /etc/bind/reverse.europa.net.db

Reiniciamos el servicio de bind

sudo systemctl restart bind9

Verificamos el estado del servicio

sudo systemctl status bind9

Maquina Cliente

Vaya a cualquier máquina cliente y agregue nuestra nueva dirección IP del servidor DNS en el archivo /etc/resolv.conf.

sudo nano /etc/resolv.conf

Utilice el comando dig para verificar la búsqueda directa

dig www.europa.net

Confirme la búsqueda inversa con el comando dig.

dig -x 192.168.0.3
